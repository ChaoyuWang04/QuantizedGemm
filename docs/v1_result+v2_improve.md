Key Issues:
- S2: BLOCK_M=64 但 M=16, 75% 算力浪费 → 需要 autotune
- S3/S4: BLOCK 太小, Tensor Core 利用率低 → 需要增大 BLOCK
- 全部 Roof% < 25%, 优化空间巨大

v2 优化清单（按 ROI 排序）
═══════════════════════════════════════════════════

★ 优先级 1: @triton.autotune
  - 投入: 1 小时
  - 预期收益: S2 从 0.29x → 1.5-2x, S3/S4 提升 2-3x
  - 为什么 #1: S2 暴露的 BLOCK 失配问题是所有形状的共性问题

  优先级 2: 调大 BLOCK_M/N/K
  - 投入: 半小时
  - 预期收益: S3/S4 Roof% 从 14-23% → 40-60%
  - 为什么不#1: autotune 会自动覆盖这个

  优先级 3: 矩阵 tile 的 super-grouping (swizzle)
  - 投入: 1-2 小时
  - 预期收益: 大形状 +10-20% L2 cache 命中
  - 为什么不#1: 复杂度高, autotune 收益更直接

  优先级 4: split-K (K 维度并行)
  - 投入: 2-3 小时
  - 预期收益: M=1 这种瘦长形状 +30-50%
  - 为什么不#1: 只对特定形状有效

  优先级 5: scale 的预乘 (sa × sb 提前算)
  - 投入: 半小时
  - 预期收益: 1-3%
  - 为什么不#1: 收益小, 后期再做
